# Codex 临时工作区约定

`temporary/` 用于存放 Codex 在分析、开发和验证过程中产生的临时文件。它把一次性产物与项目源码分开，便于检查和清理。

## 适合存放的内容

- 临时 Python、Shell 或 PowerShell 脚本
- 测试输出、截图、日志和中间数据
- 文档草稿、排查记录和命令结果
- 不准备进入正式源码目录的实验文件

现有的 `test_outputs/` 用于保存测试生成的图片等结果。

## 使用规则

1. 不要在此保存密码、令牌、私钥或个人数据。
2. 脚本需要长期维护时，应移入 `scripts/` 或对应功能模块。
3. 正式测试应放入项目测试目录；这里只保留一次性验证产物。
4. 临时文件应使用能说明用途的名称，避免 `test1.py`、`new.md` 等模糊命名。
5. 完成任务后可以删除无复用价值的文件，但不要清理其他开发者仍在使用的内容。

## Git 规则

本地仓库通过 `.git/info/exclude` 排除了整个 `temporary/` 目录。默认情况下，目录内的新文件不会出现在 `git status` 中，也不会被普通的 `git add` 纳入提交。

只有明确要求长期保留的说明文档或复现材料才应强制加入版本控制，例如：

```bash
git add -f temporary/CODEX_WORKSPACE.md temporary/PROJECT_GUIDE.md
```

提交前应使用 `git diff --cached --name-only` 检查范围，确保没有加入测试输出、日志或大文件。
